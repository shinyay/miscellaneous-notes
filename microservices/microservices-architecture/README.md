## Description
### マイクロサービスアーキテクチャ
#### 粒度
- サービスが小さければ小さいほど
  - 相互依存関係にかかわる利点が増加する
  - 複雑度が増す
- 自律性
  - 依存関係が強すぎると、依存先の内部表現から制約をうける

#### 採用の利点
- 高い回復性
  - 問題箇所の隔離が可能
- スケールの容易性
  - プロセスレベルのスケール
- デプロイの容易性

### モデル化
- 疎結合
- 高凝集
- 時期尚早な分解
  - まずはモノリシック
  - ドメインの理解が進むにつれて徐々にマイクロサービスに分離
